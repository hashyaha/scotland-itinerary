# Scotland Highlands Itinerary

A self-contained trip planner for the Scotland Highlands trip, 30 Apr – 4 May 2025.

Generated by `src/main.rs` using [Maud](https://maud.lambda.xyz/) — a compile-time
HTML templating library for Rust. Every stop and day is a typed Rust struct; the
template iterates over them to produce clean, static HTML.

## Generate the HTML

```bash
cargo run
# → index.html
```

Requires Rust (install via https://rustup.rs).

## GitHub Pages setup (5 minutes)

1. **Create a repo** at github.com/new  
   Name it something like `scotland-trip-2025`

2. **Upload `index.html`**  
   Rename it to `index.html` first, then drag-and-drop into the repo root.

3. **Enable Pages**  
   Repo → Settings → Pages → Source: Deploy from branch → `main` / `/(root)` → Save

4. **Share the URL**  
   `https://YOUR-USERNAME.github.io/scotland-trip-2025/`  
   Drop this into the WhatsApp group. Done.

### What "editable" means on GitHub Pages

| Who          | How they edit                                                    |
|--------------|------------------------------------------------------------------|
| Trip organiser | Edit `index.html` directly on GitHub, commit → Pages auto-redeploys in ~30s |
| Group members  | Open the URL → tap any text → edits save to **their device** via localStorage |

Group members can't push changes back to the repo (that would need a backend).
For a travel trip this is totally fine: the organiser owns the master, everyone
else has their own annotated copy on their phone.

### Want true group sync without a backend?

Use the Claude.ai artifact version of this file — it uses shared storage so
all edits sync automatically between anyone who has the conversation open.

## File structure

```
scotland-itinerary/
├── Cargo.toml        # Rust project config
├── src/
│   └── main.rs       # All data + Maud templates + CSS + JS
└── index.html       # Generated — this is what you deploy
```

## Rust architecture (src/main.rs)

```
enum Dot   { Gold | Warn | Moss }     ← dot colours
enum Tag   { Drive | Hike | ... }     ← stop type + CSS class
struct Stop { id, time, tag, name, note, ... }
struct Day  { id, tab, date, title, alert?, stops }

fn render_stop(s: &Stop, day_id: u8)  → Markup
fn render_day(d: &Day, active: bool)  → Markup
fn render_tabs(days: &[Day])          → Markup
fn render_document(days: &[Day])      → Markup   ← full HTML
fn main()  → writes index.html
```

CSS uses custom properties (`--time-w`, `--tl-pad`, `--stop-gap`, `--dot-w`) so
the timeline line position is always calculated precisely from the same source of
truth as the flex layout — no magic pixel values.
